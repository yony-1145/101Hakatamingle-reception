# ユーザー種別と権限

## 認証方式

- **会員**: メール / パスワード認証
- **ゲスト**: OTP認証（メアド + 6桁コード）。Supabase Auth の OTP（One-Time Password）機能を使用。セッションは当日の登録〜チェックイン完了までの一時用（最大3時間。Supabase Auth JWT expiry で制御）
- **運営**: メール / パスワード認証 + 管理者ロール

認証基盤: Supabase Auth

## ユーザー種別

| 種別 | 認証 | できること |
| --- | --- | --- |
| ゲスト | OTP認証（メアド + 6桁コード） ※名前は自動生成 | イベント閲覧・当日参加登録（会場QRから） |
| 参加者（会員） | ログイン済み | 全機能（マイページ・参加履歴・セルフチェックイン等） |
| 運営 | ログイン済み + 管理者ロール | 管理画面（イベント作成・参加者管理・チェックイン処理） |

## ゲストの機能制限

- **事前登録**（MVPではゲスト参加は当日会場QRからのみ。事前登録は会員特典）
- **マイページ・参加履歴の永続閲覧**（OTPセッションは当日の登録〜チェックインのみ有効。セッション切れ後はアクセス不可）
- **会員アカウントへの引き継ぎ**（MVP対象外。out-of-scope.md 参照）
- **セルフチェックイン**（QRコードスキャン後の本人確認ができないため、運営による手動チェックインで対応）
- **プロフィール編集・パスワード管理**（アカウント未作成のため）

## 権限制御

- 運営ロール: `users.role = 'admin'` で区別。Row Level Security (RLS) で制御
- 運営ロールの付与: MVPでは Supabase ダッシュボードから手動実施（フェーズ2でアプリ内 UI を実装予定）
- 個人情報の方針: 取得は最小限。クレジットカード情報はアプリ側で保持しない（Stripeに委託）

## Admin向けMFA（多要素認証）

### 概要

- **対象**: `users.role = 'admin'` のみ（強制）。参加者（role = 'participant'）は任意
- **方式**: TOTP（Time-based One-Time Password）
  - Google Authenticator / Microsoft Authenticator / Authy 等で生成
  - 6桁コード / 30秒ごとに更新
- **Backup Codes**: MFA デバイス喪失時の復旧用（10コード自動生成 / 1回限り）

### ログインフロー

**初回ログイン時（admin）:**
1. メール/パスワード認証 → 成功
2. 画面自動リダイレクト: 「MFA設定が必須です」
3. QRコード表示 → スマホアプリでスキャン
4. OTP コード 6桁入力 → 確認
5. Backup Codes 表示 → ダウンロード / コピーして安全に保存
6. 設定完了 → 管理画面へアクセス可能

**通常ログイン時（admin）:**
1. メール/パスワード認証 → 成功
2. OTP入力画面 → 6桁コード入力（or Backup Code 1つ使用）
3. 検証成功 → 管理画面アクセス

### リカバリ・リセット

- **MFA デバイス喪失**: Backup Code が残っていれば Backup Code でログイン → MFA 再設定可能
- **Backup Code 全使用済み + デバイス喪失**: 別の admin が Supabase ダッシュボードから当該ユーザーの MFA をリセット

### 実装

- Supabase Auth 標準 MFA 機能（TOTP）を使用
- RLS: admin ロール login 時に mfa_verified = true を要求（Supabase Auth 標準で実装）
- 参加者は MFA 設定 UI を任意で利用可（セキュリティ向上オプション）
