# 利用フロー

各フローは「誰が・いつ・どの順で操作するか」のシーケンスを定義する。検証ルール・判定ロジックの詳細は [business-rules.md](./business-rules.md) を参照。

## 参加者フロー（会員）

### Path A: Stripe事前決済

```
ログイン/登録 → 規約同意(初回) → イベント選択 → Stripe決済 → 参加確定
                                                                    ↓
                              [イベント当日] 会場のQRをスキャン → 自動チェックイン完了 → スタッフ受付画面に通知
```

1. ログイン（未登録の場合は新規登録）
2. 規約同意（初回のみ / 写真撮影に関する同意を含む）
3. イベント一覧から参加したいイベントを選択
4. Stripe Checkoutで決済
5. 参加確定 → メール通知受信
6. イベント当日: 会場のQRをスマホでスキャン → サーバー検証通過後、自動でチェックイン完了（名前・イベント名・日程・Paid済み）→ スタッフ受付画面に「〇〇さんがチェックインしました」通知

### Path B: 当日現金払い

```
ログイン/登録 → 規約同意(初回) → イベント選択 → 現金払いで参加確定
                                                                    ↓
                              [イベント当日] 会場のQRをスキャン → 自動でスタッフ通知 → 現金受取 → チェックイン完了
```

1. ログイン（未登録の場合は新規登録）
2. 規約同意（初回のみ）
3. イベント一覧から参加したいイベントを選択
4. 当日現金払いを選択 → 参加確定（paymentStatus=unpaid）→ メール通知受信
5. イベント当日: 会場のQRをスキャン → サーバー検証で現金/unpaid判定 → 自動でスタッフ受付画面に「〇〇さんが受付待ちです」通知 → 参加者画面「受付待ち中...」表示 → スタッフが確認 → 現金受取 → チェックイン完了
   - 補完: QRがスキャンできない場合はマイページから「チェックインする」ボタン押下で同様の通知を発火

## ゲストフロー（当日登録のみ）

MVPではゲストは事前登録不可。当日会場でのQRスキャンから登録するフローのみ提供。

```
会場でQRスキャン（未ログイン） → ログイン画面 → 「ゲストとして登録」
→ メアド入力 → OTP送信 → 6桁コード入力 → 認証完了（一時セッション）
→ 規約同意 → 当日現金払いで参加確定
                                                            ↓
              自動で「チェックインする」発火 → スタッフ受付画面に通知 → 現金受取 → チェックイン完了
```

1. QRをスキャン → 未ログインのためログイン画面表示
2. 「ゲストとして登録」リンク → メアド入力画面
3. メアド入力 → OTPコード送信 → 6桁コード入力 → 認証完了（最大3時間の一時セッション）
4. 規約同意（写真撮影同意含む）を入力 ※名前は自動生成
5. 決済（当日現金払いのみ。stripe_onlyイベントはゲスト非対応のためボタン非表示）→ 参加確定
6. 登録完了直後に自動で「チェックインする」処理を発火 → スタッフ受付画面に「〇〇さんが受付待ちです」通知 → スタッフが確認 → 現金受取 → チェックイン完了
7. キャンセルは原則不可（当日登録直後の取消は受付スタッフへ口頭で伝達 → 運営が管理画面から削除）

## 運営フロー

1. **イベント作成・公開時の設定:**
   - 決済方式（stripe_only / stripe_and_cash）
   - 定員・返金期限・キャンセル期限
   - 担当スタッフを3名程度割り当て（event_staff）
2. **参加申込の通知**をメールで受信（運営共有アドレス）
3. **管理画面**で参加者一覧・支払いステータス・参加者詳細を確認
4. **イベント当日:**
   - 会場にチェックイン用QRを掲示
   - Stripe事前決済の会員: QRスキャン → 自動チェックイン → スタッフ受付画面に通知（操作不要）
   - ゲスト・当日現金ユーザー: 参加者がQRスキャン or 「チェックインする」→ 通知 → 現金受取（必要に応じて）→ チェックイン完了
   - 定員到達時は「受付終了」と表示。キャンセル枠が出ると自動で再受付
   - 必要に応じて `allowOverCapacity` で定員超過を許可
5. **イベント終了後:**
   - 最終的なチェックイン数を確認
   - 売上を確認（Stripe + 現金）
   - 監査ログで運営操作を確認可能
6. **イベントキャンセル時**（開催前に中止する場合）:
   - 管理画面から `events.status = 'cancelled'` に変更
   - 全参加登録を自動キャンセル / Stripe決済済み参加者は自動全額返金 / キャンセル通知メールを自動送信
   - 詳細は [business-rules.md - イベント運営都合キャンセル](./business-rules.md#イベント運営都合キャンセル) を参照

## イベント種別とフローの対応

- **事前決済限定イベント（stripe_only）**: 会員のみ対象。Stripe決済が完了していないとセルフチェックインできない。ゲスト非対応
- **現金払い可イベント（stripe_and_cash）**: 会員はStripe事前決済 or 当日現金、ゲストは当日会場登録時の現金のみ
