# MVP機能要件

機能の能力（capability）を一覧化。検証ルール・判定ロジック・運用ルールの詳細は [business-rules.md](./business-rules.md)、シーケンスは [flows.md](./flows.md)、テーブル定義は [data-schema.md](./data-schema.md) を参照。

## 参加者向け

### 認証・アカウント

- ログイン / 新規登録（メール+パスワード）
- メール認証（新規登録時にメールアドレスの本人確認を実施）
- パスワードリセット（メールによるパスワード再設定）
- 規約同意（写真撮影同意含む / バージョン管理 / 日英2バージョン）→ 詳細は [規約同意ポリシー](./business-rules.md#規約同意ポリシー)
- マイページ（要ログイン）
  - プロフィール表示・編集
  - 参加予定イベント
  - 参加回数・履歴
- 多言語対応（日英）→ 詳細は [多言語対応ルール](./business-rules.md#多言語対応ルール)

### 新規登録の入力項目

- **必須**: email / password / name
- **任意**（マイページで後から入力可能）: birthDate / attribute / nationality / residenceCountry / residenceCity / preferredLanguage
- 段階的な入力を許容し、新規登録の離脱を減らす

**住居情報の入力UI:**

- `residenceCountry`: ISO国リストからプルダウン選択（日本がデフォルト候補）
- `residenceCity`: 自由記述テキスト
- `residenceCountry = 'JP'` の場合のみ、人口上位5市 + その他の**クイック選択チップ**を表示:
  - 福岡市 / 北九州市 / 久留米市 / 飯塚市 / 大牟田市 / その他
  - チップを押すとテキストフィールドに都市名が自動入力
  - 「その他」を押すとテキストが空になり自由入力モードに

### イベント参加

- イベント一覧・詳細閲覧（ゲストも可。残り枠数・地図リンク表示）
- イベント参加登録
  - 会員: ログイン状態から登録（Stripe決済 or 当日現金）
  - ゲスト: 当日会場QRから登録のみ（OTP認証 + 規約同意 + 名前自動生成 + 当日現金払い）
- ゲスト登録（会場QRからのみ / OTP認証完了後にメアド本人確認済 + 規約同意で登録 / stripe_onlyイベントは非対応でボタン非表示）
- イベント参加キャンセル（会員のみ。開始時刻まで可能。ゲストは原則キャンセル不可、運営の管理画面操作で対応）→ 詳細は [キャンセル・返金判定ロジック](./business-rules.md#キャンセル返金判定ロジック)
- 当日飛び込み参加（`endDateTime`まで受付。定員到達時は運営の手動オーバーライドで受付可能）
- 決済: Stripe Checkout 事前決済 / 当日現金払い

### チェックイン・参加状況

- セルフチェックイン（会員のStripe事前決済のみ対応 / QRスキャンで自動完了）→ 詳細は [チェックイン検証ルール](./business-rules.md#チェックイン検証ルール)
- 参加状況の表示（マイページ / 既存フィールドから導出）→ 詳細は [参加状況の表示](./business-rules.md#参加状況の表示マイページ)
- 支払いステータス表示: 未払い / 支払い済み / 返金済み

### ゲストOTP認証（当日登録の一時用）

- 会場QRスキャン後の登録フローでメアド本人確認のみに使用
- メールアドレス入力 → 6桁OTPコード送信 → コード入力で認証完了（最大3時間の一時セッション）
- 用途: 規約同意 → 名前自動生成 → 当日現金払い登録 → 自動チェックイン処理 のフロー完結まで
- セッション切れ後の再アクセスは不可（マイページなし）

### メール通知

- 参加登録確認・リマインド受信 → 詳細は [メール通知ルール](./business-rules.md#メール通知ルール)

## 運営向け

### イベント管理

- 一覧 / 詳細 / 作成 / 編集 / 複製 / 受付停止 / キャンセル / **QRを再発行**
- イベントキャンセル → 詳細は [イベント運営都合キャンセル](./business-rules.md#イベント運営都合キャンセル)
- QR再発行 → 詳細は [QR再発行](./business-rules.md#qr再発行流出時の対応)
- 返金ポリシー設定（イベントごとに返金可否）
- 定員管理
  - 定員到達時: 「満員のため受付を締め切りました」（キャンセル枠で自動再受付）
  - 運営による手動受付停止: 「受付を締め切りました」（定員到達とは別表記）
  - 定員超過許可: 運営の手動オーバーライドで定員超過受付可

### 受付業務

- **受付専用画面**: イベント当日のワンストップ画面。全 admin がアクセス可能（複数運営者が同時操作可能）
  - チェックイン用QRコードの表示
  - **参加者リスト**: イベント登録者全員（チェックイン済み・未チェックイン両方）。Supabase Realtime でリアルタイム更新
    - 各参加者: 名前 / メアド一部表示（ゲスト向け） / 支払い方法 / チェックインステータス / チェックイン完了ボタン（未チェックインのみ）
  - **リアルタイム通知**: QRスキャン時にトースト通知（「〇〇さんが受付待ちです」「〇〇さんがチェックインしました」）→ タップでリスト内の該当参加者にジャンプ
  - 名前/メアド検索による手動チェックイン（ゲスト含む）
  - **アクセス権**: `users.role = 'admin'` なら全員アクセス可。event_staff 割当は操作履歴の名前付与用（アクセス制限ではなく運営チーム識別用）
- リアルタイム受付状況: 登録数 / チェックイン済み / 未チェックインをリアルタイム表示

### 参加者管理

- 参加者一覧（イベントごと）
- 支払いステータス管理（未払い / 支払い済み / 返金済み / 返金失敗中）
- 参加キャンセル処理
- 全ユーザー登録者一覧
- 未払いリマインド（一括メール送信）
- 売上確認（イベントごと / `SUM(feeAtRegistration WHERE paymentStatus='paid' AND refundStatus IS NULL OR refundStatus='failed')` で算出。返金完了済みは除外）
  - **返金失敗中** (`paymentStatus='paid' AND refundStatus='failed'`): 売上に含める。管理画面で「手動返金待ち」として追跡。Stripeダッシュボードリンク付き
- 現金返金の手動記録（返金日時・金額を入力 → `registrations.refundedAt` / `refundedAmount` に保存）

### スタッフ割当

- 各イベントに担当スタッフ（管理者）を3名程度割り当て（`event_staff`）
- 割り当てられたスタッフは管理画面・受付専用画面で確認可能

### 自動処理

- イベント自動クローズ（`endDateTime`経過後に自動でclosed）
- メール通知受信（参加申込時に運営共有アドレスへ送信）
- **Stripe支払い状況定期同期**（毎時Cron）: Webhook未受信時の支払い確認漏れを防止 → 詳細は [Stripe決済確認フロー](./business-rules.md#stripe決済確認フロー)

### 監査・ロール・セキュリティ

- 運営ロール管理: MVPでは Supabase ダッシュボードから手動付与（フェーズ2でアプリ内UI予定）
- **Admin向けMFA設定**: 初回ログイン時に自動リダイレクト。TOTP QRコード表示 → アプリ登録 → Backup Codes生成・ダウンロード → 詳細は [users.md - Admin向けMFA](./users.md#admin向けmfa多要素認証)
- 運営操作の監査ログ: 重要操作を `admin_audit_logs` に自動記録 → 詳細は [data-schema.md - admin_audit_logs](./data-schema.md#運営操作の監査ログ-admin_audit_logs)
  - 記録対象: 手動チェックイン / 支払いステータス変更 / 現金返金記録 / 定員オーバーライド / 受付停止・再開 / 参加キャンセル（運営操作）/ イベントキャンセル / QR再発行
  - MVPでは閲覧UIなし、Supabase ダッシュボードから直接クエリで確認
